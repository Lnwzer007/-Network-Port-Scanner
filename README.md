# -Network-Port-Scanner

สคริปต์ Python สำหรับตรวจสอบพอร์ตเชื่อมต่อ (Port Scanning) บนระบบเครือข่ายและอุปกรณ์ Internet of Things (IoT) เช่น กล้องวงจรปิด (IP Camera), เครื่องบันทึกภาพ (DVR/NVR)
โดยมีฟังก์ชันการทำงานหลักในการส่งทราฟฟิกผ่าน Proxy Server (SOCKS4, SOCKS5, HTTP) เพื่อประโยชน์ในการทดสอบระบบรักษาความปลอดภัยจากภายนอกเครือข่าย
และการปกปิดสิทธิ์การเข้าถึงในฐานะผู้ตรวจสอบระบบ

สคริปต์นี้เลือกใช้เทคนิคการสแกนรูปแบบ **TCP Connect Scan** ซึ่งระบุการใช้งานผ่านฟังก์ชัน `socket.connect_ex()` ในภาษา Python 
หลักการทำงานจะอ้างอิงตามมาตรฐานโปรโตคอล TCP (RFC 793) โดยการสร้างการเชื่อมต่อแบบครบกระบวนการ 3-Way Handshake ดังนี้

1. **[SYN]** - เครื่องผู้สแกนส่งแพ็กเก็ต SYN (Synchronize) ไปยังพอร์ตเป้าหมาย
2. **[SYN-ACK]**  หากพอร์ตนั้นเปิดอยู่ อุปกรณ์ปลายทางจะตอบกลับด้วย SYN-ACK
3. **[ACK]**  เครื่องผู้สแกนจะส่งแพ็กเก็ต ACK เพื่อยืนยันการเชื่อมต่อที่สมบูรณ์

สคริปต์นี้แก้ปัญหาการเปิดเผย IP จริงผ่านเทคนิค **Socket Overriding** โดยการใช้โมดูล `PySocks` เข้ามาควบคุมโครงสร้างพื้นฐานของระบบผ่านคำสั่ง
python
socks.set_default_proxy(PROXY_TYPE, PROXY_HOST, PROXY_PORT)
socket.socket = socks.socksocket

ติดตั้งโมดูล
pip install PySocks

ตัวแปรในสคริปต์
การตั้งค่าตัวแปรในสคริปต์ (Configuration Variables)
PROXY_HOST: หมายเลข IP หรือ Domain Name ของ Proxy Server ที่ต้องการใช้งาน
PROXY_PORT: หมายเลขพอร์ตการให้บริการของ Proxy Server
PROXY_TYPE: รูปแบบโปรโตคอลการเข้ารหัส โดยเลือกจากคลาส socks
target_ip: หมายเลข IP ที่ต้องการสแกน
ports_to_scan: Array ของหมายเลขพอร์ตเครือข่ายที่ต้องการส่งคำขอเชื่อมต่อเพื่อตรวจสอบสถานะ

Port คืออะไร? (What is a Port?)
พอร์ต (Port) ถ้าให้อธิบายง่ายๆ มันคือ “หมายเลขช่องทาง” หรือเปรียบเหมือน “ประตูห้อง” ในบ้าน
สมมติว่า IP Address คือที่อยู่บ้าน (บอกว่ากล้องตั้งอยู่ไหนในเน็ตเวิร์ก) ตัว Port ก็จะเป็นเลขห้องในบ้านหลังนั้น เพราะกล้อง 1 ตัว มันทำหน้าที่หลายอย่างพร้อมกัน
ข้อมูลมันเลยต้องวิ่งแยกห้องกันเพื่อไม่ให้ตีกันเองครับ เช่น:
เดินเข้าประตู 80 = ไปโผล่หน้าเว็บเพื่อตั้งค่ากล้อง เดินเข้าประตู 554 = ไปดึงเอาไฟล์วิดีโอภาพสดออกมาดู
อุปกรณ์หนึ่งตัวจะมีพอร์ตได้ตั้งแต่เลข 1 ไปจนถึง 65535 การสแกนพอร์ตก็คือการไล่เคาะดูทีละประตูว่า ห้องไหนเปิดไฟอยู่และทำงานอะไรอยู่ข้างใน

# ลิสต์พอร์ตยอดนิยมสำหรับสแกนกล้อง CCTV และอุปกรณ์ Network ทั่วไป
ports_to_scan = [
    21, 22, 23, 80, 443, 554, 1935, 37777, 8000, 8080, 81, 82, 83, 88, 5000, 9000
]

ห้องสำหรับเข้าหน้าเว็บไปตั้งค่ากล้อง
Port 80 / 443 (HTTP/HTTPS): ประตูหลักสำหรับเข้าหน้าเว็บไปตั้งค่ากล้อง ห้องนี้จะเสี่ยงสุดๆ ถ้าเราไม่ยอมเปลี่ยนรหัสผ่านเริ่มต้นที่แถมมากับโรงงาน หรือปล่อยให้เฟิร์มแวร์เก่าจนมีช่องโหว่ให้คนอื่นแอบล็อกอินเข้ามา

Port 8080 / 81 / 82 / 83 / 88: พอร์ตทางเลือกที่ช่างมักจะเปลี่ยนหนีพอร์ตมาตรฐาน เพื่อหลบไม่ให้โดนคนอื่นแอบมาสุ่มสแกนหน้าเว็บเจอได้ง่ายๆ

ห้องสำหรับส่งสัญญาณภาพและเสียงสด (Video Streaming)
Port 554 (RTSP): พอร์ตหากินของกล้องแทบทุกตัวในโลกเอาไว้ส่งภาพสด ถ้าห้องนี้ไม่ตั้งรหัสผ่านล็อกเอาไว้ ใครก็สามารถแอบมาดึงภาพวิดีโอจากกล้องเราไปดูผ่านโปรแกรมทั่วไปอย่าง VLC ได้ทันที

Port 1935 (RTMP): พอร์ตสำหรับส่งภาพสดข้ามเน็ตเวิร์ก มักจะเจอในกล้องรุ่นใหม่ๆ ที่เราสั่งให้มันไลฟ์สตรีมภาพตรงขึ้นพวกแพลตฟอร์มออนไลน์ได้เลย

ห้องเฉพาะของแต่ละแบรนด์ (Vendor SDK)
Port 8000: ประตูเฉพาะของแบรนด์ Hikvision เอาไว้เชื่อมต่อดูกล้องผ่านแอปในมือถือหรือพวกโปรแกรมบริหารจัดการในคอมพิวเตอร์

Port 37777: ประตูเฉพาะของแบรนด์ Dahua ทำหน้าที่รับส่งไฟล์ภาพและรับคำสั่งควบคุมกล้องผ่านแอป DMSS

Port 9000 / 5000: มักจะเจอในกล้องจีนทั่วไป หรือกล้อง IP Camera สำเร็จรูปที่ต้องใช้แอปเฉพาะของตัวเองในการเชื่อมต่อ

ห้องควบคุมระบบและส่งข้อมูลหลังบ้าน (Management & File Transfer)
Port 23 (Telnet): ประตูสั่งการยุคเก่าที่ไม่มีการเข้ารหัสข้อมูลเลย ถือเป็นห้องที่อันตรายที่สุดเพราะเสี่ยงต่อการถูกบอทเน็ตเจาะเข้ามาฝังมัลแวร์ในกล้อง

Port 22 (SSH): ประตูสำหรับเข้าหลังบ้านแบบปลอดภัย มีการเข้ารหัสอย่างดี ส่วนใหญ่จะเปิดไว้ให้ช่างเทคนิครีโมทเข้ามาแก้ระบบลึกๆ

Port 21 (FTP): ประตูส่งไฟล์ เปิดไว้เวลาเราตั้งค่าให้กล้องส่งไฟล์ภาพหรือวิดีโอที่มันแคปได้ตอนเจอคนเดินผ่าน ขึ้นไปเก็บไว้บนเซิร์ฟเวอร์ส่วนกลาง
